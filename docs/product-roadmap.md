## Build Philosophy

1. **Each phase ships something usable.** No phase leaves the app broken. After any phase, you can demo what you've built.
2. **Infrastructure before features.** The first phase sets up everything so feature work in later phases is fast and clean.
3. **Magic moment first.** The core value proposition — analyze, design, A/B compare, export — works as early as possible. Everything else builds on top.
4. **Test as you go.** Don't save testing for the end. Each task includes verification.
5. **Progressive enhancement.** Start with the simplest working version, then layer on polish.
6. **Review before proceeding.** Push each completed phase as a PR and let a review agent (e.g. CodeRabbit) review it before starting the next phase. External review catches issues the coding agent won't flag.
7. **Sound quality is the gatekeeper.** Before moving past Phase 2, get producer feedback on audio output quality. If it's not studio-grade, stop and fix it before building more UI.
8. **Local-first always.** Every feature must work offline. The license server is the only external dependency, and it's Phase 4.

### Session Tips

- **Verify after each phase:** Run the app after completing a phase to confirm everything works before moving on.
- **Review before next phase:** Push a PR for review and wait for approval before starting the next phase. This catches issues early.
- **Don't skip tasks:** Tasks are ordered intentionally. Skipping creates dependencies and broken features.
- **One task at a time:** Focus on one task per message to the agent. Let it complete fully before asking about the next.

### Phase Completion Checklist

After completing all tasks in a phase:
- [ ] All tasks marked `- [x]`
- [ ] App launches and runs without crashes
- [ ] All phase features work end-to-end (test the workflows described in the phase goal)
- [ ] No TypeScript errors (`npm run build` succeeds)
- [ ] Code is formatted and linted (`npm run lint` succeeds, if configured)
- [ ] Commit message includes phase name and task count
- [ ] PR is descriptive and includes phase goal in the body

### Verification Guide

**Phase 0:** App launches, three-panel layout renders, Tailwind tokens applied, SQLite initialized.
**Phase 1:** Load WAV file, hear audio playback, adjust knobs, hear effects in real-time, waveform updates visually.
**Phase 2:** Load file or preset, see analysis metadata, adjust effects, A/B compare, export as WAV/MP3.
**Phase 3:** Create a kit with 4 sounds, export the kit as a folder with named WAV files.
**Phase 4:** Activate a test key, confirm premium tier is enabled, deactivate, confirm revert to free.
**Phase 5:** Test all error messages, keyboard shortcuts, accessibility (Tab navigation), build for native platform.
**Phase 6:** Batch analyze a folder, save and load custom presets, import a preset file.
**Phase 7:** Full feature set working, 80+ presets available, 10 languages supported, CI/CD builds automatically.

### Troubleshooting Common Issues

**Audio not playing after loading a file:**
- Verify AudioContext is initialized (may need user interaction first).
- Check that AudioBufferSource is connected to the destination.
- Confirm the decoded AudioBuffer has non-zero length.

**Waveform not updating during knob adjustments:**
- Verify AnalyserNode is connected to the output of the effects chain.
- Check that `requestAnimationFrame` is being called regularly.
- Confirm FFT data is being read and passed to the waveform renderer.

**IPC calls failing with undefined:**
- Verify the handler is registered in the main process via `ipcMain.handle()`.
- Confirm the handler name matches the call from the renderer.
- Check that preload script exposes the handler via `contextBridge`.

**Knob drag feels sluggish:**
- Reduce the debounce delay for effects updates (should be < 50ms).
- Check that audio updates are not being throttled.
- Profile with DevTools to identify bottlenecks (CPU, GPU, memory).

**Presets not loading in the browser:**
- Verify the presets table is populated on app startup.
- Confirm file paths in the presets table are correct (use absolute paths or relative to `public/`).
- Check that audio files exist at the specified paths.

**License activation failing:**
- Verify the license server is running and reachable (check `LICENSE_SERVER_URL` in environment).
- Confirm the serial key is correct and has activations remaining.
- Check that device fingerprint is being hashed correctly (SHA-256).
